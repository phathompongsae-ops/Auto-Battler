# Combat System Decisions

เอกสารนี้สรุปกติกาที่แพตช์ `patches/combat-targeting.patch` ใช้ และรายการที่ยังต้องตัดสินใจก่อนขยายระบบต่อสู้ต่อ

## กติกาที่แพตช์กำหนดแล้ว

1. **ตัวประชิดตีทแยงได้**
   - ใช้ Chebyshev distance สำหรับตรวจระยะต่อสู้
   - ตัวประชิดจึงถือว่าศัตรูใน 8 ช่องรอบตัวอยู่ติดกัน
   - การเดิน A* ยังเดิน 4 ทิศเพื่อป้องกันการตัดมุมผ่านยูนิต

2. **เป้าหมายหลักคือศัตรูที่เข้าถึงได้จริง**
   - ตัวประชิดหาเส้นทางไปยังช่องว่างรอบเป้าหมาย ไม่เดินเข้าไทล์ของศัตรู
   - ศัตรูที่ไม่มีช่องเข้าถึงจะไม่ถูกเลือกเป็นเป้าหมายหลัก

3. **ล็อกเป้าหมายชั่วคราว**
   - ยูนิตเก็บ `target` ไว้ ไม่เลือกใหม่ทุกเฟรม
   - ตรวจสอบใหม่ทุกประมาณ 0.4 วินาที หรือเมื่อเป้าหมายตาย/ไปไม่ถึง

4. **เกณฑ์ตัดสินเมื่อมีหลายเป้าหมาย**
   1. ระยะเส้นทางสั้นกว่า
   2. เปอร์เซ็นต์ HP ต่ำกว่า
   3. แถวบนก่อน
   4. คอลัมน์ซ้ายก่อน

   กติกานี้ทำให้ผล deterministic มากขึ้นและทดสอบบาลานซ์ซ้ำได้

5. **โจมตีระยะไกลใช้ projectile**
   - ดาเมจเกิดเมื่อลูกกระสุนเดินทางถึงเป้าหมาย
   - ถ้าเป้าหมายตายก่อน ลูกกระสุนจะถูกลบและไม่เปลี่ยนเป้าหมายกลางทาง

6. **Beast Lord เป็นโจมตีระยะกลาง ระยะ 3 ช่อง**
   - โจมตีปกติด้วยพลังจากคทาในระยะกลาง ไม่ใช่แครี่ระยะไกลเต็มตัว
   - มังกรวิญญาณที่เรียกออกมาในอนาคตควรเป็นยูนิตประชิดแยกต่างหาก
   - ไม่มีข้อมูลยืนยันว่าต้องรวมร่างก่อนจึงโจมตีไกล

## สิ่งที่ยังต้องตัดสินใจ

### A. Projectile และแนวยิง

- ยิงทะลุยูนิตฝ่ายเดียวกันและสิ่งกีดขวางได้หรือไม่
- ต้องมี line of sight หรือไม่
- กระสุนตามเป้าหมายแบบ homing หรือบินไปยังตำแหน่งเดิม
- ถ้าเป้าหมายตายระหว่างทาง กระสุนหาย เปลี่ยนเป้าหมาย หรือระเบิดตรงจุดเดิม
- เวทบางชนิดควรเป็น hitscan, projectile หรือพื้นที่ AoE

**ค่าแนะนำสำหรับเวอร์ชันแรก:** ยิงทะลุเพื่อนได้, ไม่มี line of sight, projectile ตามเป้าหมาย, เป้าหมายตายแล้วกระสุนหาย

### B. Aggro และการเปลี่ยนเป้าหมาย

- ยูนิตควรเปลี่ยนไปตีศัตรูที่กำลังโจมตีตนเองหรือไม่
- Tank จะมี Taunt หรือเพิ่มน้ำหนักการถูกเลือกเป็นเป้าหมายหรือไม่
- Assassin ควรเริ่มจากตัวใกล้สุด หรือกระโดดหาแนวหลัง
- เมื่อถูกโจมตีจากระยะไกล ตัวประชิดควรหันไปไล่ผู้ยิงหรือคงเป้าหมายเดิม

**ค่าแนะนำ:** ตัวทั่วไปล็อกเป้าหมายเดิม; Tank มี Taunt ผ่านสกิล; Assassin ใช้ targeting policy เฉพาะคลาส

### C. การชนและการเคลื่อนที่

- ยูนิตฝ่ายเดียวกันสลับตำแหน่งกันได้หรือไม่
- อนุญาตให้สองยูนิตจองช่องเดียวกันชั่วคราวหรือไม่
- เมื่อทางถูกบล็อกควรรอ เปลี่ยนเป้าหมาย หรือหาทางใหม่ทันที
- บอสขนาด 2 ช่องจะครอบครองพื้นที่จริง 2×2 หรือเป็นเพียงภาพใหญ่

**ค่าแนะนำ:** ไม่ซ้อนช่อง, ไม่สลับผ่านกัน, คำนวณเส้นทางใหม่เมื่อบล็อก, บอสใหญ่ควรมี footprint จริงภายหลัง

### D. ระยะโจมตี

- ค่าระยะควรวัดแบบ Chebyshev ทุกคลาสหรือเฉพาะ melee
- ระยะของ AoE วัดแบบวงกลม รูปเพชร หรือสี่เหลี่ยม
- Sniper ระยะ 5 ควรมีระยะขั้นต่ำหรือไม่
- ตัวยิงไกลควรถอยเมื่อศัตรูประชิดเข้าถึงหรือไม่

**ค่าแนะนำ:** ใช้ Chebyshev สำหรับโจมตีทั้งหมดในเวอร์ชัน grid; Sniper ไม่มีระยะขั้นต่ำในเวอร์ชันแรก; ยังไม่เพิ่มระบบถอยอัตโนมัติ

### E. ความเร็วโจมตีและจังหวะดาเมจ

- ดาเมจประชิดเกิดตอนเริ่มแอนิเมชัน กลางแอนิเมชัน หรือเฟรม strike
- projectile ถูกปล่อยตอนเฟรมใด
- ถ้าโดน stun ระหว่าง windup การโจมตีถูกยกเลิกหรือไม่
- ความเร็วโจมตีมีเพดานสูงสุดเท่าไร

**ค่าแนะนำ:** ดาเมจ/ปล่อยกระสุนที่เฟรม strike, ยกเลิกเมื่อโดน hard crowd control, จำกัดความเร็วโจมตี 3 ครั้งต่อวินาที

### F. ระบบสกิลและมานา

ยังไม่มีข้อกำหนดกลางสำหรับ:

- มานาเริ่มต้นและมานาที่ได้เมื่อโจมตี/ถูกโจมตี
- สกิล interrupt ได้หรือไม่
- ระยะและรูปทรง AoE
- เป้าหมายของ heal/buff/debuff
- Summon ใช้ช่องบนกระดานและนับจำนวนยูนิตหรือไม่
- Summon หายเมื่อผู้เรียกตายหรือไม่

### G. สถานะผิดปกติ

ต้องกำหนดกติกาการซ้อนของ Slow, Weaken, Stun, Silence, Burn, Poison และ Shield:

- ซ้อนค่าแรงหรือซ้อนเวลา
- รีเฟรชระยะเวลาหรือสร้างหลาย instance
- มี diminishing returns หรือไม่
- บอสต้านทานสถานะกี่เปอร์เซ็นต์

### H. Determinism และ Random Seed

ระบบ Duelist crit และ Trickster weaken ยังใช้ `Math.random()` ทำให้ผลแต่ละรอบไม่เหมือนกัน ควรตัดสินใจว่าจะ:

- ยอมให้สุ่มอิสระ
- ใช้ seeded RNG ต่อรัน
- บันทึก seed เพื่อ replay/debug

**ค่าแนะนำ:** ใช้ seeded RNG ต่อรันก่อนเริ่มทำระบบ replay และบาลานซ์จริง

## ลำดับงานถัดไปที่แนะนำ

1. นำแพตช์ targeting ไปใช้และทดสอบกรณีทางตัน/ยูนิตรุมเป้าหมายเดียว
2. แยก combat logic ออกจาก `autochess.html` เป็นโมดูล
3. เพิ่ม test scenario แบบ deterministic สำหรับ melee/ranged/path blocking
4. กำหนด projectile/line-of-sight และ aggro policy
5. ออกแบบ mana/skill pipeline
6. เพิ่ม footprint สำหรับบอสตัวใหญ่
7. เพิ่ม seeded RNG และ combat log สำหรับวิเคราะห์บาลานซ์
